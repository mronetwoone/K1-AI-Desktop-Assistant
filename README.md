# 基于进迭时空 K1 的桌面 AI 伴学机器人及端侧大模型本地部署优化

这是比赛初赛版本的工程仓库。当前版本主要完成了桌面伴学机器人的核心闭环：K1 本地姿态识别、ASRPRO 离线语音交互、STM32 表情屏联动、学习计时和环境温湿度查询。

项目的整体思路比较直接：摄像头负责看坐姿，K1 负责跑姿态模型和做状态判断，ASRPRO 负责听指令和播报，STM32 表情屏负责显示表情并桥接环境传感器。端侧大模型部分在当前版本中按“状态上下文封装 + 语义策略接口”的方式接入整体架构，用来承接后续的本地语义理解、个性化提醒和学习建议生成。

## 已实现功能

- K1 端运行 YOLOv8n-pose 姿态检测模型
- 根据鼻尖、双肩关键点判断驼背、高低肩和复合异常
- 支持异常连续帧确认、恢复确认和同一异常锁存，减少重复提醒
- ASRPRO 本地离线语音控制：开始检测、关闭检测、检测状态、当前温湿度
- 学习计时：开始学习、查询学习时长、停止学习
- STM32 表情屏联动：loving、happy、sad、thinking、angry 等表情
- DHT11 温湿度数据读取与语音播报
- K1、ASRPRO、STM32 之间使用轻量串口协议通信

## 硬件连接

| 模块 | 连接方式 | 说明 |
|---|---|---|
| 摄像头 | 接 K1，设备号 `/dev/video20` | 采集用户上半身画面 |
| ASRPRO | UART 接 K1，默认 `/dev/ttyS10`，9600 bps | 离线语音识别和播报 |
| STM32 表情屏 | USB-TTL 接 K1，常见 `/dev/ttyACM0`，115200 bps | 表情显示和环境数据返回 |
| DHT11 | 接 STM32 | 温湿度采集 |
| 喇叭 | 接 ASRPRO | 语音输出 |

ASRPRO 与 K1 串口连接时需要 TX/RX 交叉，并保证共地。

## K1 端编译与运行

进入 K1 端代码目录：

```bash
cd src/k1
mkdir -p build
cd build
cmake ..
make -j4
```

运行前确认串口权限：

```bash
sudo chmod 666 /dev/ttyS10
sudo chmod 666 /dev/ttyACM0
```

运行：

```bash
FACE_UART_DEVICE=/dev/ttyACM0 ./pose_detector
```

也可以使用脚本：

```bash
./run.sh
```

模型默认路径在 `main.cpp` 中：

```cpp
/home/lu/Horse/Q_S/model/yolov8n-pose-320.onnx
```

如果模型放在其他目录，直接修改 `model_path` 即可。

## ASRPRO 语音命令

| 语音命令 | 作用 |
|---|---|
| 小助手 | 唤醒语音交互 |
| 开始检测 | 打开坐姿检测播报 |
| 关闭检测 | 关闭坐姿检测播报 |
| 检测状态 | 查询当前检测开关状态 |
| 当前温湿度 / 播报环境状态 | 请求 K1 读取温湿度并播报 |
| 开始学习 | 开始学习计时 |
| 学多久了 | 播报当前学习时长 |
| 停止学习 | 停止计时并播报本次学习时长 |

ASRPRO 工程文件在 `src/asrpro/voice_module.hd`，修改识别词或播报内容后需要在天问 Block 中重新生成模型并下载。

## 串口协议

协议整理在 `docs/uart_protocol.md`。实际调试中最常用的是下面几类：

- K1 向 ASRPRO 发送 `AA xx 55`，触发坐姿提醒播报
- ASRPRO 向 K1 发送 `AB 20 55`，请求温湿度
- K1 向 STM32 发送 `happy\r\n`、`sad\r\n`、`angry\r\n` 等文本命令切换表情

## 当前状态

初赛版本已经完成核心功能演示。后续主要优化方向：

1. 固定串口设备名，减少每次调试前手动 `chmod` 的步骤；
2. 优化外壳内部走线和散热；
3. 增加学习记录保存和日报统计；
4. 在 K1 端进一步接入本地小参数大模型，用统一状态上下文生成更自然的提醒话术和学习建议。
